# Prompt: โมเดล 3D โกดังสเกลจริง (สำหรับ Claude Design)

Prompt ด้านล่างใช้สั่ง Claude Design ให้สร้าง HTML ไฟล์เดียว + Three.js แสดงโกดัง 3D
สเกลจริง แบ่งโซน วางสินค้าตามขนาด+จำนวน stock จริง และมีมุมกล้องเลียนแบบ CCTV
เพื่อเทียบกับภาพที่ AI ใช้นับ stock

ชื่อ field ใน DATA ตรงกับระบบ stock เดิม (`code/widthCm/lengthCm/heightCm/stock/noLayDown`,
Zone `productIds`) เพื่อให้สลับจากข้อมูลตัวอย่างเป็นข้อมูลจริงได้โดยไม่ต้องแก้ logic

---

## คัดลอกส่วนนี้ไปวางใน Claude Design

```
สร้างไฟล์ HTML ไฟล์เดียว เปิดในเบราว์เซอร์แล้วรันได้ทันที (โหลด Three.js + OrbitControls จาก CDN ผ่าน importmap ไม่ต้อง build) เพื่อแสดง "โกดังสินค้า 3D ตามขนาดจริง" สำหรับใช้วางแผนพื้นที่และเทียบกับภาพกล้องวงจรปิด ตามสเปกนี้

## 1) หน่วยและสเกล (สำคัญที่สุด — ต้องสเกลจริง)
- 1 unit ใน Three.js = 1 เมตร
- ขนาดโกดัง/โซนกำหนดเป็นเมตร, ขนาดสินค้ากำหนดเป็นเซนติเมตร แล้วหาร 100 ตอน render
- มีพื้น (floor) + grid ทุก 1 เมตร + ป้ายบอกระยะเป็นเมตรที่ขอบ เพื่อตรวจสเกลด้วยตาได้

## 2) โครงสร้างข้อมูล (วาง object ก้อนเดียวไว้บนสุดของไฟล์ ให้แก้ง่าย และใช้ชื่อ field ตามนี้เป๊ะ เพื่อให้สลับเป็นข้อมูลจริงจากระบบ stock ได้โดยไม่แตะ logic)

const WAREHOUSE = { widthM, lengthM, heightM };
// ขนาดโกดังจริงจะใส่ทีหลัง — ตอนนี้ใส่ค่าตัวอย่าง เช่น 20 x 30 x 6 เมตร
// ทำ comment ชัด ๆ ว่า "<<< ใส่ขนาดจริงตรงนี้"

const ZONES = [
  {
    id, name, note,
    origin: { x, z },   // มุมเริ่มของพื้นที่โซนบนพื้น (เมตร)
    size:   { w, l },   // กว้าง x ยาว ของโซน (เมตร)
    color,              // สีประจำโซน
    productIds: [...],  // id สินค้าที่อยู่ในโซนนี้ (ตรงกับ Zone.productIds ในระบบเดิม)
    camera: { position:[x,y,z], target:[x,y,z], fov }  // มุมกล้องเลียนแบบ CCTV ของโซน (optional)
  }
];

const PRODUCTS = [
  {
    id, code, name, nameT, brand,
    stock,            // จำนวนชิ้นในสต็อกจริง
    widthCm, lengthCm, heightCm,
    cubicM,           // optional: ถ้ามีให้ใช้แทน w*l*h
    sizeClass,        // "S"|"M"|"L"|"XL" ใช้ fallback ถ้าไม่มีขนาด (กำหนด m³ default ต่อ class)
    noLayDown,        // true = ห้ามวางตะแคง (เช่น ตู้เย็น) ต้องตั้งตามแกนสูงเสมอ
    unit
  }
];

ใส่ข้อมูลตัวอย่างที่สมจริง: โกดัง 1 หลัง, 3–4 โซน, แต่ละโซนมีสินค้าหลายรายการ, ค่า stock หลากหลาย (บางอันหลักหน่วย บางอันหลักร้อย), มีสินค้าแบบ noLayDown อย่างน้อย 1 ตัว เพื่อทดสอบการจัดเรียงและการนับเต็มพื้นที่

## 3) การวางสินค้าในโซน (ตามสเกลจริง + ตามจำนวน)
- แต่ละสินค้าวาดเป็นกล่องตามขนาดจริง widthCm x lengthCm x heightCm
- แสดง "จำนวนจริง" ด้วยการเรียง/ซ้อนกล่องในพื้นที่ของโซนนั้น: เรียงตามแนวกว้างจนเต็มแถว -> ขึ้นแถวใหม่ตามแนวยาว -> เต็มชั้นแล้วซ้อนขึ้นตามความสูง (จนถึงเพดานโซน/โกดัง)
- เคารพ noLayDown: ถ้า true ห้ามหมุนวางตะแคง
- ถ้า stock เยอะมาก (เช่น > ~200 ชิ้น) ให้วางเป็น "กอง/พาเลทตัวแทน" ก้อนใหญ่ก้อนเดียวที่ปริมาตรรวมใกล้เคียงของจริง แล้วติด label "x{stock}" เพื่อกันเครื่องค้าง
- สินค้าต้องอยู่ในกรอบพื้นที่ของโซนเท่านั้น ถ้าล้นเกินพื้นที่ให้เปลี่ยนสี/ขึ้นเตือน

## 4) มุมมองและการใช้งานหลัก
(ก) ดูผังตามโซน
- แต่ละโซนมีพื้น/กรอบสีของตัวเอง + ป้ายชื่อโซนลอยอยู่ด้านบน
- มี panel รายชื่อโซน: คลิกแล้วโฟกัส+ไฮไลต์เฉพาะโซนนั้น (โซนอื่นจางลง) และ filter ให้เห็นเฉพาะสินค้าในโซนนั้น
- ปุ่มสลับ "ดูทั้งหมด / ดูทีละโซน"

(ข) วัดความจุ/ความเต็ม (fill)
- คำนวณปริมาตรสินค้ารวมในโซน เทียบกับปริมาตรพื้นที่โซน -> แสดง % เต็ม
- ระบายสีแถบสถานะตามระดับ (เขียว -> เหลือง -> แดง) ให้เห็นปุ๊บรู้ว่าโซนไหนแน่น/ว่าง

(ค) เทียบกับภาพกล้องวงจรปิด (สำคัญ — เอาไว้เทียบกับภาพที่ AI ใช้นับ stock)
- แต่ละโซนตั้งมุมกล้องเลียนแบบมุม CCTV จริง (จาก ZONES[].camera)
- ปุ่ม "มุมกล้องโซนนี้": snap กล้อง 3D ไปยังมุมเดียวกับกล้องวงจรปิด เพื่อเทียบภาพ render กับภาพจริง
- ถ้าทำได้: โหมดเทียบข้างกัน (ซ้าย = ภาพ 3D มุมกล้อง, ขวา = ช่องว่างสำหรับวางภาพถ่าย CCTV จริงทีหลัง / อัปโหลดรูปมาทาบ)

## 5) Interaction
- OrbitControls: หมุน/แพน/ซูมด้วยเมาส์
- คลิกสินค้า -> popup: code, ชื่อ (nameT แล้วตามด้วย name), brand, stock + unit, ขนาด WxLxH, ปริมาตรต่อชิ้น และจำนวนชั้นที่วางได้
- toggle: เปิด/ปิด label, grid, แสดงเฉพาะโซนที่เลือก
- legend: สีประจำโซน + สเกลความเต็ม

## 6) ข้อกำหนดทางเทคนิค
- HTML ไฟล์เดียวจบ ใช้ Three.js + OrbitControls ผ่าน CDN (unpkg/jsdelivr, ESM importmap)
- แยกโครงโค้ดชัดเจนพร้อม comment: [DATA บนสุด] / [สร้าง scene+แสง+พื้น] / [build โซน] / [build+วางสินค้าตามจำนวน] / [UI/ปุ่ม]
- performance: ใช้ InstancedMesh หรือ merge geometry เมื่อกล่องเยอะ
- responsive เต็มจอ + ปุ่ม UI ลอยที่มุมจอ

เริ่มจากข้อมูลตัวอย่างก่อน: เปิดมาต้องเห็นโกดัง 3–4 โซน สินค้าจัดเรียงเต็มพื้นที่ตามจำนวน หมุนดูได้ กดดูทีละโซนได้ และกด "มุมกล้องโซนนี้" เพื่อเทียบกับ CCTV ได้ทันที
```

---

## สิ่งที่ต้องเติมทีหลัง (ตอนสลับเป็นข้อมูลจริง)
1. `WAREHOUSE` = ขนาดโกดังจริง (กว้าง/ยาว/สูง เมตร)
2. `ZONES[].origin/size` = ผังโซนจริง + `ZONES[].camera` = มุม CCTV จริงของแต่ละโซน
3. `PRODUCTS` = export ข้อมูลจริงจากแอป (field ตรงกันอยู่แล้ว) แทน sample
